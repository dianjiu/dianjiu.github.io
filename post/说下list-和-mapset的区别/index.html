说下List 和 Map、Set的区别

从存储结构分析

List和Set都是继承自Collection接口的，用于存储单列数据的集合

Map是用于存储双列数据的集合

从存储特点分析

List存储的数据是有序的，且可重复

Map存储的数据是无序的，且Key不可以重复

Set存储的数据是无序的，且不可以重复（在Set集合中的元素的存储位置是由HashCode决定的，所以位置是固定的，但是这个存储位置用户是不可控的，所以对用户来说，Set集合是无序的）

从实现类分析

List

List的实现类有LinkedList、ArrayList、Voctor

LinkedList基于链表实现的，链表的内存是散乱的，每一个元素在存储本身内存地址的同时还要维护下一个元素的地址，链表增删块，查找慢。

ArrayList基于数组实现的，非线程安全的，效率高，便于索引，不便于增删操作

Voctor基于数组实现的，线程安全的，效率低

Map

Map的实现类有HashTable、HashMap、LinkedHashMap、TreeMap

HashTable基于Hash表的接口实现，线程安全的，低效的，不支持null键和null值

HashMap基于Hash表的接口实现，线程不安全，高效，支持null键和null值

LinkedHashMap是HashMap的子类，保存了记录的插入顺序

TreeMap基于SortMap接口，能把保存记录根据键排序，默认是按键的升序排序

Set

Set的实现类有HashSet、LinkedHashSet、TreeSet

HashSet基于HashMap实现，底层使用HashMap保存所有数据，不允许存储重复数据，父类AbstractSet重写了hashCode()和equals()方法。

LinkedHashSet基于LinkedHashMap实现，同时又继承了HashSet。

TreeSet基于TreeMap实现，底层使用TreeMap保存所有数据，不允许存储重复数据，和HashSet有相同的父类AbstractSet,父类重写了hashCode()和equals()方法
